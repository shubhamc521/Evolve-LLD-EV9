# Types of Design Patterns

1. Creational - Talks about creation of an object
- When to create the object (Lazy vs Eager)
- How many instances to create (One or  many)
- Which type to create (based on runtime conditions)

- Creational design pattern controls HOW, WHEN and WHICH objects are created

- Five Design Patterns solves different problems

Singleton = ONE
Factory = CREATE based on type
Abstract Factory = CREATE family
Builder = Build complex
Prototype = CLONE
